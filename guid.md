# План за имплементация: URL база данни с автоматично инкрементиране

## 1. Обобщение на изискванията

### 1.1. Проблем
Потребителят има множество отворени табове с URL адреси за сваляне на изображения и постоянно трябва да навигира между тях и интерфейса на приложението, за да копира ръчно всеки URL. Това е неефективно.

### 1.2. Решение
Създаване на JSON файл (`urls.json`) който служи като локална база данни с URL адреси. Приложението да може да:
- Съхранява списък с URL адреси
- Извършва CRUD операции (добавяне, изтриване, промяна)
- Зарежда избран URL директно в полето за обработка
- Автоматично инкрементира URL адреса след успешно сваляне

---

## 2. Архитектура на промените

### 2.1. Нови/променени файлове

| Файл | Тип промяна | Описание |
|------|-------------|----------|
| `urls.json` | **НОВ** | JSON база с URL адреси |
| `url_manager.py` | **НОВ** | Модул за CRUD операции с JSON базата |
| `app.py` | **ПРОМЯНА** | Нови Flask маршрути за API endpoints |
| `templates/index.html` | **ПРОМЯНА** | Нов UI компонент за управление на списъка |
| `scraper.py` | **ПРОМЯНА** | Логика за автоматично инкрементиране след сваляне |

### 2.2. Структура на `urls.json`

Максимално опростена структура - само масив от URL низове:

```json
[
  "https://content3.erosberry.com/wowgirls.com/0017/00.jpg",
  "https://content3.erosberry.com/wowgirls.com/0018/00.jpg",
  "https://cdn.example.com/gallery/0001/photo_001.jpg"
]
```

## 3. Подробен план стъпка по стъпка

### СТЪПКА 1: Създаване на `url_manager.py` (НОВ модул)

**Цел:** Модул за управление на JSON базата с URL адреси.

**Подстъпки:**

1.1. **Създаване на клас `UrlManager`:**
   - Конструктор: приема път до JSON файла (`urls.json`)
   - При инициализация: ако файлът не съществува, създава празен масив `[]`

1.2. **Метод `load_urls()`:**
   - Зарежда и връща списъка от URL адреси от JSON файла
   - Обработка на грешки (ако файлът е повреден)

1.3. **Метод `save_urls(urls)`:**
   - Записва списъка от URL адреси в JSON файла
   - Форматиране с отстъп за четимост

1.4. **Метод `add_url(url)`:**
   - Проверка за дублиране чрез **нормализиране на URL-а** (виж Стъпка 1.5)
   - Ако URL-ът не съществува (по основна директория), го добавя
   - Връща `(success, message)`

1.5. **Метод `get_base_url(url)` - ключова логика за проверка на дублиране:**
   - Взема URL адреса
   - Извлича **основната директория** (протокол + домейн + първите 2 нива на пътя)
   - Пример: `https://content3.erosberry.com/wowgirls.com/0017/00.jpg` → `https://content3.erosberry.com/wowgirls.com`
   - Пример: `https://cdn.example.com/gallery/0001/photo_001.jpg` → `https://cdn.example.com/gallery`
   - **Алгоритъм:**
     1. Премахва протокола (`https://`)
     2. Разделя останалата част по `/`
     3. Взема първите 3 части: `[домейн, ниво1, ниво2]`
     4. Ако има по-малко от 3 части, взема всички налични
     5. Връща `протокол + части.join('/')`

1.6. **Метод `is_duplicate(url)`:**
   - Използва `get_base_url(url)` за нормализиране
   - Сравнява с всички съществуващи URL-и в списъка (също нормализирани)
   - Връща `True`/`False`

1.7. **Метод `delete_url(index)`:**
   - Изтрива URL на посочената позиция
   - Връща `(success, message)`

1.8. **Метод `update_url(index, new_url)`:**
   - Променя URL на посочената позиция
   - Проверка за дублиране (без да сравнява със себе си)
   - Връща `(success, message)`

1.9. **Метод `increment_url(index)` - ключова логика за автоматично инкрементиране:**
   - Взема URL-а на посочената позиция
   - Намира **последното число в пътя преди името на файла**
   - **Алгоритъм:**
     1. Разделя URL-а по `/`
     2. Последният елемент е името на файла (напр. `00.jpg`)
     3. Предпоследният елемент е директорията (напр. `0017`)
     4. Ако предпоследният елемент съдържа само цифри, инкрементира числото
     5. Ако предпоследният елемент съдържа цифри + текст, намира числовата част и я инкрементира
     6. Запазва броя на водещите нули (padding)
     7. Конструира новия URL със старата стойност на файла
   - Пример: `https://content3.erosberry.com/wowgirls.com/0017/00.jpg` → `https://content3.erosberry.com/wowgirls.com/0018/00.jpg`
   - Записва промяната в JSON файла
   - Връща новия URL

### СТЪПКА 2: Промяна на `app.py` - нови Flask маршрути

**Цел:** Добавяне на API endpoints за CRUD операции и интеграция със scraper-а.

**Подстъпки:**

2.1. **Импорт на `UrlManager`:**
   ```python
   from url_manager import UrlManager
   ```

2.2. **Инициализация на мениджъра:**
   ```python
   url_manager = UrlManager('urls.json')
   ```

2.3. **Нов маршрут `GET /api/urls`:**
   - Връща целия списък от URL адреси като JSON
   - Формат: `{"urls": [...], "count": N}`

2.4. **Нов маршрут `POST /api/urls/add`:**
   - Приема JSON с поле `url`
   - Извиква `url_manager.add_url(url)`
   - Връща `{"success": true/false, "message": "...", "urls": [...]}`

2.5. **Нов маршрут `POST /api/urls/delete`:**
   - Приема JSON с поле `index`
   - Извиква `url_manager.delete_url(index)`
   - Връща `{"success": true/false, "message": "...", "urls": [...]}`

2.6. **Нов маршрут `POST /api/urls/update`:**
   - Приема JSON с полета `index` и `url`
   - Извиква `url_manager.update_url(index, url)`
   - Връща `{"success": true/false, "message": "...", "urls": [...]}`

2.7. **Промяна на маршрут `/download`:**
   - След успешно завършване на цикъла (в `scraper.py`), scraper-ът трябва да комуникира обратно кои URL-и от списъка са били използвани
   - Добавяне на параметър `url_index` (по избор) - индекса на URL-а в списъка
   - Ако `url_index` е подаден и свалянето е успешно, извиква `url_manager.increment_url(url_index)`
   - Включва информация за инкрементирането в SSE съобщението

### СТЪПКА 3: Промяна на `scraper.py` - поддръжка на URL индекс

**Цел:** scraper-ът да може да приема индекс на URL от списъка и да връща информация дали е използван такъв.

**Подстъпки:**

3.1. **Промяна на `download_images()`:**
   - Добавяне на параметър `url_index` (по избор, `None` по подразбиране)
   - Ако `url_index` не е `None`, scraper-ът знае че работи с URL от списъка

3.2. **Промяна на `process_single_url()`:**
   - Добавяне на параметър `url_index`
   - След успешно завършване на цикъла (свалени са всички итерации или е достигнат 404):
     - Ако `url_index` не е `None`, добавя информация в `downloaded_files_list` за инкрементиране

3.3. **Логика за идентифициране на числото за инкрементиране в пътя:**

     - Взема предпоследния елемент (директорията преди файла)
     - Търси число в този елемент с regex
     - Ако има число, го инкрементира, запазвайки padding
     - Конструира новия URL
   - Тази функция се използва и от `UrlManager.increment_url()`

**Подстъпки:**

     - Автоматично обновява списъка с URL адреси

4.7. **CSS стилове за новия компонент:**

## 4. Детайлна спецификация на ключовите алгоритми

### 4.1. Алгоритъм за `get_base_url(url)` - проверка за дублиране

```
Вход: "https://content3.erosberry.com/wowgirls.com/0017/00.jpg"

1. Премахни протокола: "content3.erosberry.com/wowgirls.com/0017/00.jpg"
2. Раздели по '/': ["content3.erosberry.com", "wowgirls.com", "0017", "00.jpg"]
3. Вземи първите 3 части: ["content3.erosberry.com", "wowgirls.com", "0017"]
4. Ако има по-малко от 3, вземи всички
5. Върни: "https://content3.erosberry.com/wowgirls.com/0017"

Вход: "https://cdn.example.com/gallery/0001/photo_001.jpg"

1. Премахни протокола: "cdn.example.com/gallery/0001/photo_001.jpg"
2. Раздели по '/': ["cdn.example.com", "gallery", "0001", "photo_001.jpg"]
3. Вземи първите 3 части: ["cdn.example.com", "gallery", "0001"]
4. Върни: "https://cdn.example.com/gallery/0001"
```

**Важно:** Сравнението за дублиране се прави между `get_base_url(new_url)` и `get_base_url(existing_url)` за всеки съществуващ URL.

### 4.2. Алгоритъм за `increment_path_number(url)` - инкрементиране на число в пътя

```
Вход: "https://content3.erosberry.com/wowgirls.com/0017/00.jpg"

1. Раздели по '/': ["https:", "", "content3.erosberry.com", "wowgirls.com", "0017", "00.jpg"]
2. Предпоследен елемент: "0017"
3. Провери дали съдържа число: да, "0017"
4. Инкрементирай числото: 0017 → 0018 (запази padding = 4)
5. Замени предпоследния елемент: "0018"
6. Конструирай нов URL: "https://content3.erosberry.com/wowgirls.com/0018/00.jpg"
7. Върни новия URL

Вход: "https://cdn.example.com/gallery/0001/photo_001.jpg"

1. Раздели по '/': ["https:", "", "cdn.example.com", "gallery", "0001", "photo_001.jpg"]
2. Предпоследен елемент: "0001"
3. Инкрементирай: 0001 → 0002
4. Нов URL: "https://cdn.example.com/gallery/0002/photo_001.jpg"
```

**Важно:** Ако предпоследният елемент съдържа текст + цифри (напр. `album0017`), regex трябва да намери числовата част и да я инкрементира: `album0017` → `album0018`.

## 7. Бележки

- JSON файлът `urls.json` се създава автоматично при първото добавяне на URL
- При грешка в JSON файла (повреден), се създава резервно копие и се започва с празен списък
- Всички промени в списъка се отразяват веднага в JSON файла
- Интерфейсът поддържа тъмна тема
- Максимум 10 реда се виждат наведнъж в списъка, останалите са достъпни чрез скрол
