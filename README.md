# image-downloader

Flask web application for automatic downloading of sequential images from URLs with auto-increment support

## Функционалности

- Автоматично сваляне на последователни изображения
- Управление на URL списък (добавяне, изтриване, редакция)
- Автоматично инкрементиране на URL номера
- Real-time прогрес на свалянето
- Уеб интерфейс

## Технологии

- Python 3.8+
- Flask 3.1.3
- Requests 2.32.3

## Инсталация

### Windows

#### Метод 1: Клониране на Git репозиторито (препоръчително)

1. **Отворете Command Prompt или PowerShell:**
   - Натиснете `Win + R`
   - Въведете `cmd` или `powershell`
   - Натиснете Enter

2. **Навигирайте до желаната директория:**
   ```cmd
   cd C:\Users\вашето_име\Documents
   ```
   Или към друга директория, където искате проекта.

3. **Клонирайте репозиторито:**
   ```cmd
   git clone https://github.com/ваш-username/image-downloader.git
   ```

4. **Влезте в директорията на проекта:**
   ```cmd
   cd image-downloader
   ```

5. **Стартирайте приложението:**
   ```cmd
   start_app.bat
   ```

6. **Готово!** Приложението ще се отвори автоматично в браузъра.

---

#### Метод 2: Ръчно изтегляне (без Git)

1. **Изтеглете като ZIP:**
   - Отворете репозиторито в GitHub
   - Натиснете зеленото бутон "Code"
   - Изберете "Download ZIP"
   - Разпакетете ZIP файла в желана директория

2. **Отворете Command Prompt в директорията:**
   - Отивате в директорията с разпакетираните файлове
   - В адресната лента на Explorer въведете `cmd` и натиснете Enter

3. **Стартирайте приложението:**
   ```cmd
   start_app.bat
   ```

## Автоматични обновления

Приложението проверява за обновления при стартиране, като сравнява датата на последния локален commit с последния commit в GitHub repository-то.

### Как работят обновленията:

1. **При стартиране на приложението**, системата автоматично проверява GitHub за нови промени
2. **Ако има нова версия**, се показва банер с информация за обновлението
3. **Потребителят може да избере**:
   - "🔄 Обнови сега" - автоматично извлича новите промени чрез `git pull`
   - "✕ По-късно" - отхвърля обновлението за сега

### Изисквания за обновления:

- ✅ Продуктът трябва да е клониран с Git (не работи с ZIP изтегляне)
- ✅ Git трябва да е инсталиран и в PATH
- ✅ Потребителят трябва да има права за запис в директорията на проекта
- ✅ Интернет връзка за проверка и обновяване

### Конфигурация на GitHub репозиторито:

По подразбиране системата е конфигурирана да проверява репозиторито:
- **Owner**: DeyanShahov
- **Repository**: image-downloader
- **Branch**: main

За да промените тези настройки, редактирайте `version_checker.py`:
```python
GITHUB_OWNER = "ваш_username"
GITHUB_REPO = "ваш_repo"
```

### Ръчна проверка за обновления:

Можете да тествате системата, като стартирате:
```cmd
python version_checker.py
```

Това ще провери за обновления и ще ви предложи да ги инсталирате, ако има такива.

### Отказване от обновления:

Ако не желаете автоматични обновления:
- От менюто с обновление, натиснете "По-късно"
- Обновлението ще бъде предложено отново при следващо стартиране
